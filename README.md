##  Research Question

> *Can a computer system replicate the split-second risk judgment an experienced Cairo taxi driver makes when approaching a pedestrian вАФ predicting whether they will dart into traffic before the vehicle reaches them?*

This thesis compares three methods for predicting pedestrian crossing intention using demographic and behavioral features, framed through real-world driver risk assessment rather than academic observation.

### Equipment
| Item | Details |

### Location

### Recording Protocol
- Continuous driving footage across 6 sessions вЖТ **V1 through V6**
- Morning and afternoon lighting conditions
- Pedestrian tracks extracted per-session
- Only frames where a full pedestrian silhouette was visible were retained

##  Step 2 вАФ Annotation with CVAT

All video versions were uploaded to **CVAT (Computer Vision Annotation Tool)** for manual labeling.

### Two annotation passes were performed:

**Pass 1 вАФ Action Labels** (`frameslabels.csv`)
Each pedestrian bounding box per frame was labeled as one of:
- `Walking` вАФ moving along the pavement
- `Standing` вАФ stationary at or near the kerb
- `Crossing` вАФ stepping into or traversing the road

**Pass 2 вАФ Demographic & Behavioral Attributes** (`crossing_annotations.csv`)
For each pedestrian track, a human annotator assigned:

| Attribute | Values |
|---|---|
| `age` | child / adult / senior |
| `gender` | male / female |
| `distracted` | yes / no |
| `relativeposition` | near / middle / far |
| `judgmentcalls` | handsignal / nohandsignal |
| `dresscodestyle` | casual / formal |
| `location` | onsidewalk / notonsidewalk |
| `fashionpersonality` | minimalist / extravagant |
| `weight` | normalweight / overweight |
| `crossed` | **crossed / waited** вЖР ground truth label |

> These 9 features encode the same cues an experienced driver reads in a fraction of a second.

### Why each feature matters (taxi driver logic)
- **Children** act impulsively вЖТ higher crossing risk regardless of traffic gaps
- **Elderly** tend to be cautious вЖТ lower crossing probability
- **Distracted** pedestrians miss signals and gaps вЖТ high risk
- **Not on sidewalk** = already in road = committed to crossing
- **Hand signal** = explicitly negotiating with traffic = about to cross

---

##  Step 3 вАФ Behavioral Augmentation

To address class imbalance and improve model generalization, a **behavior-aware augmentation** strategy was applied to the annotation CSV:

- Synthetic rows generated by perturbing attribute combinations
- Augmentation rules based on real behavioral logic (e.g. distracted + near kerb вЖТ more likely to cross)
- Saved as: `crossing_annotations_behavior_aware_aug.csv`

This ensured the ML models were not biased by the majority class and learned genuine demographic-to-behavior mappings rather than frequency artifacts.

---

##  Step 4 вАФ Three Classification Methods

### Method 1 вАФ Structured Annotation + Voting Ensemble (M1)
- 9 demographic features label-encoded
- Voting ensemble: **Random Forest + Gradient Boosting + SVM**
- Baseline вАФ represents what structured human annotation alone can predict

### Method 2 вАФ Annotation вЖТ RoBERTa Embeddings (M2)
- Annotation fields converted to natural language sentences
- Encoded with **RoBERTa** (`all-roberta-large-v1`) вЖТ 768-dim embeddings
- Classified with an **MLP** on top of PCA-reduced embeddings
- Tests whether richer text representations outperform categorical encoding

### Method 3 вАФ GPT-4o Vision with Taxi Driver Prompt (M3)
- Raw pedestrian crop image sent directly to **GPT-4o**
- System prompt frames the model as an experienced **Egyptian taxi driver**
- GPT-4o extracts behavioral cues (momentum, gaze, posture, road awareness) in natural language
- Description embedded with `text-embedding-3-large` вЖТ 3072-dim vector
- Classified with a **Random Forest** on PCA-reduced embeddings
- **No human annotation required at inference time**

---

##  Step 5 вАФ LSTM Action Classifier

To avoid running the expensive 3-model pipeline on everyone in the scene, a **Bidirectional LSTM** was trained to first classify each tracked pedestrian's motion:

| Class | Description |
|---|---|
| `Walking` | Moving along pavement вАФ no crossing intent assessment needed |
| `Standing` | Stationary вАФ no crossing intent assessment needed |
| `Crossing` | **Triggers full 3-model risk assessment** |

**Input features** (8 per frame): `cx, cy, w, h, dx, dy, speed, aspect_ratio`
**Architecture**: 2-layer Bidirectional LSTM вЖТ LayerNorm вЖТ Dropout вЖТ Linear head
**Best validation accuracy**: **86.2%**

