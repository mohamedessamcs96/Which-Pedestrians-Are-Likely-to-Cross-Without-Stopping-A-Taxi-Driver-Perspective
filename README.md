## 🎯 Research Question

> *Can a computer system replicate the split-second risk judgment an experienced Cairo taxi driver makes when approaching a pedestrian — predicting whether they will dart into traffic before the vehicle reaches them?*

This thesis compares three methods for predicting pedestrian crossing intention using demographic and behavioral features, framed through real-world driver risk assessment rather than academic observation.

---

## 📹 Step 1 — Dataset Recording

### Equipment
| Item | Details |
|---|---|
| **Camera** | iPhone 11 (4K, 30fps) |
| **Vehicle** | Speranza A113 |
| **Mounting** | Dashboard-mounted, driver POV |

### Location
**Ismailia Road, Cairo, El Sharqia, Egypt**
Mixed urban traffic — intersections, pedestrian crossings, roadside walkways. Chosen to reflect real Egyptian driving conditions including informal crossing behavior, mixed-age pedestrians, and variable traffic density.

### Recording Protocol
- Continuous driving footage across 6 sessions → **V1 through V6**
- Morning and afternoon lighting conditions
- Pedestrian tracks extracted per-session
- Only frames where a full pedestrian silhouette was visible were retained

---

## 🏷️ Step 2 — Annotation with CVAT

All video versions were uploaded to **CVAT (Computer Vision Annotation Tool)** for manual labeling.

### Two annotation passes were performed:

**Pass 1 — Action Labels** (`frameslabels.csv`)
Each pedestrian bounding box per frame was labeled as one of:
- `Walking` — moving along the pavement
- `Standing` — stationary at or near the kerb
- `Crossing` — stepping into or traversing the road

**Pass 2 — Demographic & Behavioral Attributes** (`crossing_annotations.csv`)
For each pedestrian track, a human annotator assigned:

| Attribute | Values |
|---|---|
| `age` | child / adult / senior |
| `gender` | male / female |
| `distracted` | yes / no |
| `relativeposition` | near / middle / far |
| `judgmentcalls` | handsignal / nohandsignal |
| `dresscodestyle` | casual / formal |
| `location` | onsidewalk / notonsidewalk |
| `fashionpersonality` | minimalist / extravagant |
| `weight` | normalweight / overweight |
| `crossed` | **crossed / waited** ← ground truth label |

> These 9 features encode the same cues an experienced driver reads in a fraction of a second.

### Why each feature matters (taxi driver logic)
- **Children** act impulsively → higher crossing risk regardless of traffic gaps
- **Elderly** tend to be cautious → lower crossing probability
- **Distracted** pedestrians miss signals and gaps → high risk
- **Not on sidewalk** = already in road = committed to crossing
- **Hand signal** = explicitly negotiating with traffic = about to cross

---

## 🔬 Step 3 — Behavioral Augmentation

To address class imbalance and improve model generalization, a **behavior-aware augmentation** strategy was applied to the annotation CSV:

- Synthetic rows generated by perturbing attribute combinations
- Augmentation rules based on real behavioral logic (e.g. distracted + near kerb → more likely to cross)
- Saved as: `crossing_annotations_behavior_aware_aug.csv`

This ensured the ML models were not biased by the majority class and learned genuine demographic-to-behavior mappings rather than frequency artifacts.

---

## 🧠 Step 4 — Three Classification Methods

### Method 1 — Structured Annotation + Voting Ensemble (M1)
- 9 demographic features label-encoded
- Voting ensemble: **Random Forest + Gradient Boosting + SVM**
- Baseline — represents what structured human annotation alone can predict

### Method 2 — Annotation → RoBERTa Embeddings (M2)
- Annotation fields converted to natural language sentences
- Encoded with **RoBERTa** (`all-roberta-large-v1`) → 768-dim embeddings
- Classified with an **MLP** on top of PCA-reduced embeddings
- Tests whether richer text representations outperform categorical encoding

### Method 3 — GPT-4o Vision with Taxi Driver Prompt (M3)
- Raw pedestrian crop image sent directly to **GPT-4o**
- System prompt frames the model as an experienced **Egyptian taxi driver**
- GPT-4o extracts behavioral cues (momentum, gaze, posture, road awareness) in natural language
- Description embedded with `text-embedding-3-large` → 3072-dim vector
- Classified with a **Random Forest** on PCA-reduced embeddings
- **No human annotation required at inference time**

---

## 🏃 Step 5 — LSTM Action Classifier

To avoid running the expensive 3-model pipeline on everyone in the scene, a **Bidirectional LSTM** was trained to first classify each tracked pedestrian's motion:

| Class | Description |
|---|---|
| `Walking` | Moving along pavement — no crossing intent assessment needed |
| `Standing` | Stationary — no crossing intent assessment needed |
| `Crossing` | **Triggers full 3-model risk assessment** |

**Input features** (8 per frame): `cx, cy, w, h, dx, dy, speed, aspect_ratio`
**Architecture**: 2-layer Bidirectional LSTM → LayerNorm → Dropout → Linear head
**Best validation accuracy**: **86.2%**

---

## 🎬 Step 6 — Full Video Pipeline

```
Input Video (iPhone 11 dashcam footage)
        ↓
YOLOv8m — Person detection (conf=0.40, every frame)
        ↓
Bidirectional LSTM — Walking / Standing / Crossing
        ↓ (Crossing only)
GPT-4o (taxi driver prompt) → description + attributes
        ↓
M1  Voting Ensemble    ──┐
M2  RoBERTa MLP        ──┼──→  Majority vote → RISK VERDICT
M3  GPT-4o RF          ──┘
        ↓
Annotated output video with per-person risk overlay
```

### Risk Output on Video

| Display | Meaning | Box colour |
|---|---|---|
| `!! WILL CROSS — HIGH RISK` | All 3 models: crossed | 🔴 Red |
| `! LIKELY TO CROSS` | 2/3 models: crossed | 🟠 Orange |
| `WILL WAIT — LOW RISK` | Majority: waited | 🟢 Green |
| `CROSSING — ASSESSING...` | LSTM confirmed, models loading | 🟡 Orange |
| Walking | Motion only | 🔵 Blue |
| Standing | Stationary | ⚫ Grey |
